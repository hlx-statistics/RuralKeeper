# RetailLog 开发规范

> 版本：1.1 · 与 `docs/01.需求文档.md` 配套 · 以当前仓库代码为准

## 1. 技术栈

| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | Vue 3.5 + Composition API | `<script setup>` |
| 语言 | TypeScript 5.8 | 构建前 `vue-tsc` 检查 |
| 构建 | Vite 6 | 配置在根目录 `vite.config.ts` |

| PWA | `vite-plugin-pwa` | `npm run build` 启用；`--mode capacitor` 时关闭 |
| 原生壳 | Capacitor 8 | Android APK；插件见下文 |

**Capacitor 插件（仅原生包使用）：**

| 插件 | 用途 |

| `@capacitor/filesystem` | 备份 JSON 写入缓存目录 |

## 3. 分层职责

1. **views**：页面布局、组合子组件、调用 composable / service，不写 IndexedDB 直连。
2. **components**：展示与局部交互，通过 props / emit 与父级通信。
3. **composables**：可复用 UI 逻辑（摄像头扫码、购物车内存、Toast）。
4. **services**：业务规则（条码唯一、库存校验、售出整单、备份恢复、分类删除联动）。
5. **db/repositories**：单表 CRUD 与索引查询，不含 UI 与业务流程。
6. **types**：与需求文档 §5 数据模型一致，全项目唯一来源。

## 4. 命名约定

- 文件：`kebab-case`（如 `goods.service.ts`）
- Vue 组件：`PascalCase`（如 `GoodsCard.vue`）
- 组合式函数：`use` 前缀（如 `useScanner`）
- 常量：`UPPER_SNAKE_CASE`
- 类型/接口：`PascalCase`，实体类型不加 `I` 前缀

## 5. 数据与 ID

- 主键：`createId()`（`crypto.randomUUID()`）。
- 条码：`goods` 表 `barcode` 唯一索引。
- 销售单未提交前仅存内存（`useSaleCart` 模块级 `ref`），确认后写入 `sales` 并扣库存、写 `logs`。
- 流水上限：`MAX_LOGS = 500`，超出时删除最旧记录（`logs.repository`）。
- 首次启动：`initAppData()` 初始化默认分类；无商品时写入 3 条示例商品。

## 6. 扫码实现

### 6.1 技术原理

`useScanner` 使用 ZXing 的 `BrowserMultiFormatReader.decodeFromConstraints`：

1. 申请后置摄像头（`facingMode: { ideal: 'environment' }`）；
2. 将 MediaStream 绑定到 `ScannerView` 内的 `<video>`；
3. 连续从视频帧解码；成功则 `result.getText()` 回调业务层。

`decodeHintType.TRY_HARDER` 已开启；`BrowserMultiFormatReader` 进程内单例复用。

### 6.2 场景区分

| 模式 | 常量 | 使用页面 | 行为 |
|------|------|----------|------|
| 销售 | `ScanMode.Sale` | `SaleView` | 常驻预览；扫到 → 加购物车 |
| 商品 | `ScanMode.Goods` | `GoodsView` 弹窗 | 见下表 `ScanIntent` |

商品页弹窗内还有三种 **扫码意图**（`ScanIntent`，非枚举常量）：

| 意图 | 入口 | 扫到后 |
|------|------|--------|
| `add` | 「扫码添加」 | 新条码 → 打开添加表单；已存在 → 询问编辑或继续扫 |
| `lookup` | 「查商品」 | 已存在 → 询问编辑；不存在 → 询问新建 |
| `form` | 表单内「扫条码」 | 填入当前表单条码栏 |

### 6.3 防抖与反馈

| 层级 | 间隔 | 位置 |
|------|------|------|
| composable | 800ms | `useScanner` 同码去重 |
| 销售页 | 1000ms | `SCAN_DEBOUNCE_MS`，`SaleView.handleSaleBarcode` |
| 反馈 | — | `useScanFlash`：全屏闪白 + `navigator.vibrate(30)` |

销售页、商品页扫码成功均会 `flash()`；离开页面或关闭弹窗须 `stop()` 释放摄像头。

`SaleView` / `GoodsView` 使用 `KeepAlive`，`onActivated` 重启扫码、`onDeactivated` 停止。

### 6.4 手动输入

- 销售页：`ManualBarcodeModal`
- 商品扫码弹窗：`prompt` 手动输入条码

> 说明：`vite-env.d.ts` 声明了 `BarcodeDetector` 类型，但 **当前实现未使用** 原生 `BarcodeDetector`，也未引入 `jsQR`。

## 8. npm 脚本

| 命令 | 说明 |
|------|------|
| `npm run dev` | 开发服务器（端口 5173） |
| `npm run build` | 类型检查 + PWA 构建 |
| `npm run build:capacitor` | 类型检查 + 原生包 Web 资源（`base: './'`，关闭 PWA） |
| `npm run preview` | 预览构建产物 |
| `npm run typecheck` | 仅类型检查 |
| `npm run cap:sync` | `build:capacitor` + `cap sync` |
| `npm run cap:android` | sync 后用 Android Studio 打开 |
| `npm run cap:run:android` | sync 后真机运行 |
| `npm run test:form` | 商品表单关闭确认逻辑（Node 脚本） |

## 9. Git 提交

- 格式：`<type>: <subject>`（type: feat / fix / refactor / docs / chore）
- 单次提交聚焦一个功能或修复

## 10. 禁止事项

- 不引入后端 API、支付相关 SDK
- 不在 `components` 内直接 `openDB`
- 不在 `repository` 内 `alert` / `confirm`（views 层可用浏览器原生对话框）
