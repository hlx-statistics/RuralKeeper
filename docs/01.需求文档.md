# 零售日志（RetailLog）需求文档

> 项目代号：RetailLog  
> 文档版本：1.1  
> 实现状态：**V1.0 已落地**（Vue 3 + Capacitor，以仓库代码为准）  
> 目标平台：Android APK（Capacitor）；亦可构建为 PWA  
> 核心理念：轻量、离线、扫码即用，适合单人打理的小商铺

---

## 1. 项目背景与目标

### 1.1 背景

小商铺（便利店、杂货店、文具店、花店等）通常没有预算购买专业 POS 机或扫码枪，店主使用手机管理商品。现有通用工具往往过于复杂或依赖网络。因此需要一个**极度轻量、单机可用、扫码建档**的商品管理系统。

日常卖货以**手机连续扫码、看合计金额、一键确认售出**为主；顾客付款由店主出示支付宝/微信收款码线下完成，**无需在 App 内对接支付**。

### 1.2 商品与条码约定（重要）

- **一条码 = 一类商品（SKU）**：例如同一品牌矿泉水，瓶身条码相同，无法在条码层面区分「第几瓶」，只代表「这是矿泉水这一款」。
- **库存以数量计**：入库时在列表选定商品后**输入本次入库数量**；售卖时扫一次码将**一件**加入当前销售单，可多次扫描累加，或在销售单里改数量。
- **不做「一物一条码」**：不维护单件「已售/未售」状态，售出即扣减对应商品的 `stock` 数量。

### 1.3 目标

- 用手机摄像头扫描商品条形码，快速查找或录入商品。
- **销售结账**：连续扫码凑单 → 实时合计 → 确认后批量扣库存。
- 商品增删改查、入库、非销售出库、分类筛选。
- 数据仅保存在手机本地，无服务器。
- 数据备份与恢复（JSON）。
- 界面适配手机触控，学习成本低。

### 1.4 支付边界（明确不做）

- 不包含支付、收款、对账、退款、与支付宝/微信 API 对接。
- 「确认售出」仅负责**记账与扣减库存**；收款由顾客扫店主线下收款码完成。

## 3. 功能需求

### 3.1 商品管理

#### 3.1.1 添加商品

- **手动添加**：顶部「➕ 手动添加」或列表右下角 FAB。
- **扫码添加**：「📷 扫码添加」→ 弹窗取景 → 识别条码 → 打开表单并预填条码。
- 必填：商品名称、条码（唯一）、售价。
- 选填：分类、进货价（`cost`）、库存（默认 0）、备注。

#### 3.1.2 编辑商品

- 列表卡片「编辑」或扫码查找到已有商品后进入编辑。
- 表单内可再次「扫条码」覆盖条码栏。
- 条码修改时检测与其他商品重复（`BARCODE_EXISTS`）。
- 关闭表单时若有未保存修改，或扫码预填了新条码，需二次确认。

#### 3.1.3 删除商品

- 编辑表单内删除，二次确认。
- 删除商品记录；历史 `sales` / `logs` 不因删除商品而自动清理（仍保留 `goodsId` 引用）。

#### 3.1.4 商品列表

- 展示：名称、条码、售价、库存、分类。
- 库存 ≤ 0 显示为空库存样式；&lt; 5 显示低库存警告色。
- 按分类筛选、按名称/条码搜索。
- 按 `updatedAt` 倒序（`goodsService.list`）。

### 3.2 销售结账（核心场景）

#### 3.2.1 进入与扫码

- 底部 **「销售」** Tab；页顶常驻摄像头预览（`ScannerView` compact）。
- 每扫一次：已存在商品 → 销售单对应行数量 +1（新行则新增）；不存在 → 询问是否跳转商品页建档（`?barcode=` 查询参数）。
- 同条码防抖：**1000ms**（`SCAN_DEBOUNCE_MS`）；底层 ZXing 另有 **800ms** 去重。
- 支持「手动输入条码」弹窗。

#### 3.2.2 销售单展示

- 品名、单价、数量、小计；底部合计件数、合计金额。
- 每行 **+ / −** 调数量，减至 0 删除行；支持清空整单（需确认）。

#### 3.2.3 确认售出

- 「确认售出」→ 二次确认件数与金额。
- 提交前校验库存；任一行不足则**整单不提交**并提示明细。
- 成功后：写 `sales`、按行扣 `stock`、写 `logs`（type=`sale`）、清空购物车。
- 不触发任何支付界面。

#### 3.2.4 销售记录

- **「我的」** 页展示最近 **20** 条（`MAX_SALE_RECORDS_DISPLAY`），点击 `alert` 查看明细。

#### 3.2.5 经营概览（近 30 天）

- **「我的」** 页顶部 **经营概览**：仅统计 **有库存**（`stock > 0`）的商品。
- **数量排名**：近 **30** 天（滚动窗口）按售出件数从高到低。
- **金额排名**：同窗口按售出金额从高到低；两榜独立排序。
- **补货提示**：窗口内有销量且当前库存 **&lt; 5** → 显示「库存偏低，建议补货」。
- **保质期提示**：全历史 **超过 60 天未售出**（从未售出也算）且有库存 → 显示「较久未售出，建议检查保质期」（仅文案提醒，无保质期字段与自动判定）。

### 3.3 库存操作

#### 3.3.1 入库

- 商品列表卡片 **「入库+」** → 已定位该商品 → `prompt` 输入数量 → `stockService.stockIn` → 写 `logs`（type=`in`）。

#### 3.3.2 出库（非销售）

- 卡片 **「出库」** → 输入数量 → `stockService.stockOut`；超库存抛出 `INSUFFICIENT_STOCK`。

#### 3.3.3 快速调整库存

- 添加/编辑表单中可直接修改 `stock` 字段（绝对值）。

> **与初版草图差异**：V1.0 入库/出库从列表卡片操作，**未**单独做「先扫码再选入库/出库」流程；扫码主要用于建档、查档与销售。

### 3.4 条形码扫描

- 后置摄像头实时扫描（ZXing `BrowserMultiFormatReader`）。
- 支持 **EAN-13、EAN-8、UPC-A、UPC-E、CODE_128**。
- 各场景均提供手动输入备用。

| 场景 | 实现位置 | 扫码后行为 |
|------|----------|------------|
| **销售页** | `SaleView` 常驻预览 | 加入购物车 +1，停留本页 |
| **扫码添加** | `GoodsView` 弹窗 `add` | 新条码 → 添加表单；已存在 → 询问编辑 |
| **查商品** | `GoodsView` 弹窗 `lookup` | 已存在 → 询问编辑；不存在 → 询问新建 |
| **表单扫条码** | `GoodsView` 弹窗 `form` | 仅填入表单条码栏 |
| **销售未找到** | 跳转商品 Tab | `?barcode=xxx` 自动打开添加表单 |

扫码成功：`useScanFlash` 短震动 + 全屏闪白（`ScanFlash` 组件）。

### 3.5 数据备份与恢复

#### 3.5.1 备份

- 「我的 → 备份数据」导出 JSON。
- 结构（`BackupData`，`version: 2`）：`goods`、`categories`、`sales`、`logs`（可选字段）。
- 文件名：`RetailLog_backup_YYYYMMDD_HHMMSS.json`。
- **PWA**：浏览器下载；**APK**：Capacitor `Filesystem` + `Share` 唤起系统保存/分享。

#### 3.5.2 恢复

- 选择 `.json` 文件 → 确认覆盖 → 清空四表后导入 → `refreshAll`。
- 恢复后若仍无分类会初始化默认分类；无商品时写入示例数据（与首次安装相同）。

> V1.0 **未**实现恢复前自动临时备份。

### 3.6 基础设置

- **分类管理**（「我的」）：预置 `食品、饮料、日用品、其他`（`DEFAULT_CATEGORIES`），可增删；删除分类时该分类下商品改为「其他」。
- **流水**：入库、非销售出库、销售确认时写入 `logs`；本地最多保留 **500** 条（`MAX_LOGS`）。
- **首次启动**：无商品时自动插入 3 条示例商品（农夫山泉等）。

## 4. 非功能需求

### 4.1 性能

- 商品 ≤ 1000 条时，列表与搜索应流畅（内存过滤）。
- 扫码识别：常规光线下约 1 秒内（依赖设备与 ZXing）。
- 销售单 ≤ 50 行时合计无明显卡顿。

### 4.2 离线可用

- 业务逻辑完全离线；IndexedDB 本地持久化。
- PWA：`vite-plugin-pwa` + Service Worker 缓存静态资源；`npm run build` 后可「添加到主屏幕」。

### 4.3 兼容性

- 目标环境：Chrome for Android 90+、Safari for iOS 14+、Capacitor WebView（Android）。
- 扫码依赖 **WebRTC `getUserMedia`** + **ZXing**；需 HTTPS 或 localhost（Capacitor 使用 `androidScheme: https`）。

### 4.4 数据安全

- 无外部传输；备份 JSON 不加密，由用户自行保管。

### 4.5 易用性

- 触控目标 ≥ 44px；删除、确认售出、覆盖恢复有确认。
- 销售页扫码区与购物车同屏可见。

## 5. 数据模型

数据库：`RetailLogDB`（IndexedDB v1）

### 商品表 `goods`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | string (UUID) | 主键 |
| barcode | string | 条形码（唯一索引） |
| name | string | 商品名称 |
| price | number | 售价（元） |
| cost | number | 进货价（可选，默认 0） |
| stock | number | 库存（整数，≥0） |
| category | string | 分类名称（默认「未分类」） |
| remark | string | 备注 |
| createdAt | number | 创建时间戳 |
| updatedAt | number | 更新时间戳 |

### 分类表 `categories`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | string | 主键 |
| name | string | 分类名称（唯一索引） |

### 销售记录表 `sales`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | string | 主键 |
| items | SaleItem[] | `{ goodsId, barcode, name, price, quantity }` |
| totalAmount | number | 合计金额 |
| totalQty | number | 合计件数 |
| createdAt | number | 售出时间戳 |

### 流水表 `logs`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | string | 主键 |
| goodsId | string | 商品 ID |
| type | `'in' \| 'out' \| 'sale'` | 入库 / 非销售出库 / 销售扣库 |
| quantity | number | 变动数量（正数） |
| beforeStock | number | 变动前库存 |
| afterStock | number | 变动后库存 |
| saleId | string? | 销售时关联 sales.id |
| remark | string? | 备注 |
| createdAt | number | 操作时间戳 |

## 6. 用户界面（当前实现）

### 6.1 底部导航（三 Tab）

| Tab | 路由 | 页面 |
|-----|------|------|
| 商品 | `#/goods` | 列表 + 搜索/筛选 + 扫码/添加 |
| 销售 | `#/sale` | 扫码预览 + 购物车 + 确认售出 |
| 我的 | `#/profile` | 备份、销售记录、分类、说明 |

`GoodsView`、`SaleView` 使用 `KeepAlive` 保持扫码状态。

### 6.2 商品页

- 顶栏：扫码添加、手动添加、查商品。
- 搜索框 + 分类下拉。
- 卡片：编辑、入库+、出库。
- 弹窗：`GoodsFormModal`（添加/编辑）、`AppModal` + `ScannerView`（扫码）。

### 6.3 销售页

- 上部：紧凑取景框 + 提示文案。
- 下部：`CartList` + `SaleFooter`（合计、清空、确认售出、手动输入）。

### 6.4 我的页

自上而下：**使用说明** → **数据备份** → **近期销售**（板块内滚动）→ **经营概览**（数量/金额 Tab，列表板块内滚动）→ **分类管理**。
- 经营概览：补货与久未售出标签。
- 备份 / 恢复（隐藏 file input）。
