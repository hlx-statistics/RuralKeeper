# Android 安装包打包指南

乡程掌柜（RuralKeeper）使用 [Capacitor 8](https://capacitorjs.com/) 将 Vue 应用封装为 **Android APK**。店主可将安装包拷到手机安装，**无需电脑常驻、无需浏览器「添加到主屏幕」**。

应用 ID：`com.ruralkeeper.app` · 显示名：**乡程掌柜**（可在 `capacitor.config.ts` 与 `android/app/src/main/res/values/strings.xml` 修改）。

## 前置条件（仅打包时需要电脑）

1. **Node.js** 18+（与日常开发相同）
2. **Android Studio**（含 Android SDK、JDK 17+）
   - 下载：https://developer.android.com/studio
   - 首次打开后按向导安装 SDK Platform 与 Build-Tools
3. 环境变量（可选）：`ANDROID_HOME` 指向 SDK 目录

## 一键同步到 Android 工程

```bash
npm install
npm run cap:sync          # 正式包：三条参考库示例商品
npm run cap:sync:debug    # 调试包：完整演示经营数据（与 npm run dev 一致）
```

该命令会：TypeScript 检查 → 以 `capacitor` / `capacitor-debug` 模式构建 Web 资源（`base: './'`，关闭 PWA）→ 复制到 `android/` 并更新原生插件。

| 同步命令 | 适用 APK | 首次空库数据 |
|----------|----------|----------------|
| `cap:sync:debug` | `assembleDebug` 调试包 | `demo-backup.json` 完整演示 |
| `cap:sync` | `assembleRelease` 发布包 | 参考库三条：`6902083893019`、`6920152439005`、`6921168520015` |

> **注意**：`cap:sync` **不会**生成 APK，只更新 `android/` 里的网页资源。要得到可安装的 `.apk`，必须再执行下面「生成 APK」中的任一步骤。打调试包前请用 `cap:sync:debug`，打发布包前请用 `cap:sync`，否则两类 APK 会共用上一次同步的网页资源。

## 生成 APK

> **`npm run cap:android` 只同步网页并打开 Android Studio，不会生成任何 `.apk` 文件。** 必须再执行下面的「命令行」或 Studio 构建。

### 命令行（推荐，路径固定）

发布包（正式示例三条商品，无 `demo-backup.json`）：

```powershell
npm run cap:apk:release
```

生成后路径：

`android/app/build/outputs/apk/release/app-release-unsigned.apk`

（约 16MB，未签名；拷到手机一般可直接安装。若要上架商店，再用 Studio **Generate Signed Bundle / APK** 打签名包。）

调试包（完整演示数据）：

```powershell
npm run cap:apk:debug
```

路径：`android/app/build/outputs/apk/debug/app-debug.apk`

等价于：

```powershell
cd android
.\gradlew.bat assembleRelease   # 或 assembleDebug
```

### 用 Android Studio

```bash
npm run cap:android:debug   # 调试包（含演示数据）
npm run cap:android         # 发布包同步后打开 Studio
```

在 Android Studio 中：

1. 等待 Gradle 同步完成
2. **调试包**：**Build → Build Bundle(s) / APK(s) → Build APK(s)** → `app-debug.apk`
3. **发布包**：**Build → Generate Signed Bundle / APK** → 选 APK、**release**、配置签名（首次需 Create new...）→ 输出 `app-release.apk`

| 类型 | 典型路径 |
|------|----------|
| 调试 | `android/app/build/outputs/apk/debug/app-debug.apk` |
| 发布（未签名） | `android/app/build/outputs/apk/release/app-release-unsigned.apk` |
| 发布（已签名） | 同上 `release/` 目录下的 `app-release.apk`（以 Studio 提示为准） |

## 发给店主安装

1. 将 `app-debug.apk` 通过微信 / U 盘 / 网盘发到手机
2. 在手机上打开文件并安装
3. 若提示「未知来源」，在系统设置中允许该来源安装应用

> **说明**：调试 APK 仅供自用或小范围试用。若要上架应用商店，需配置签名并构建 **release** 包（Build → Generate Signed Bundle / APK）。

## 命令行构建（已配置 SDK 时）

```bash
cd android
./gradlew assembleDebug
```

Windows：

```powershell
cd android
.\gradlew.bat assembleDebug
```

## 真机调试

USB 连接手机并开启开发者模式后：

```bash
npm run cap:run:android
```

## 与 PWA 版本的区别

| 项目 | PWA（浏览器） | Android APK |
|------|----------------|-------------|
| 安装方式 | 浏览器「添加到主屏幕」 | 直接安装 APK |
| 构建命令 | `npm run build` | `npm run build:capacitor` + `cap sync` |
| 日常是否依赖电脑 | 更新需重新部署静态资源 | 安装后完全独立 |
| 数据存储 | 手机本地 IndexedDB | 相同（WebView） |
| 扫码 | 浏览器 `getUserMedia` + ZXing | WebView + 相机权限（`AndroidManifest` 已声明） |
| 数据备份 | 浏览器下载 JSON | `Filesystem` 写缓存 + `Share` 系统分享/保存 |

Web 版通过 `npm run build` 部署 `dist/`；原生包勿用带 PWA Service Worker 的构建产物。

## 原生能力与权限

`android/app/src/main/AndroidManifest.xml` 已声明：

- `android.permission.CAMERA`（扫码）
- `android.hardware.camera`（非必需特性，无摄像头设备仍可安装）

备份流程（`backup.service.ts`）：导出 JSON → Base64 写入 Cache → `Share.share({ files })` 唤起系统面板，用户可选择保存到文件管理器或分享到微信等。

## 常见问题

**Q：`npm run cap:android` 提示 Unable to launch Android Studio？**  
未安装 Android Studio，或路径未被识别。安装后重试；非默认路径可设置 `CAPACITOR_ANDROID_STUDIO_PATH` 指向 `studio64.exe`。

**Q：跑了 `npm run cap:sync` 为什么找不到 APK？**  
`cap:sync` 只做同步。请用 Android Studio **Build APK**，或 `cd android` 后 `.\gradlew.bat assembleDebug`。

**Q：命令行构建提示需要 Java 17 / 当前是 Java 11？**  
Android Gradle 插件需 **JDK 17+**，系统里的 `java -version` 若是 11 会报错。任选其一：

1. **推荐**：在 `android/gradle.properties` 指定 Android Studio 自带 JBR（已预置示例路径，按本机安装目录修改）：
   ```properties
   org.gradle.java.home=D:/Program Files/Android/Android Studio/jbr
   ```
2. 或安装 [JDK 17](https://adoptium.net/)，将 `JAVA_HOME` 指向该目录后重开终端。

改完后在项目根目录重试：`npm run cap:run:android:debug`。

**Q：装到手机后能正常卸载吗？**  
可以，在系统「应用信息」中卸载即可。

**Q：标题与系统状态栏重叠（商品 / 销售 / 我的页顶到顶）？**  
需包含 `@capacitor/status-bar` 并重新 `cap:sync` 后安装。应用启动时会禁止 WebView 压到状态栏下，并为 Android 预留顶部安全区。若仍偏高/偏低，可在 `src/utils/safe-area.ts` 调整 `ANDROID_TOP_FALLBACK_PX`（默认 36）。

**Q：安装后摄像头打不开？**  
在系统设置中授予「相机」权限；进入销售页或商品扫码弹窗时会请求授权。

**Q：修改前端代码后如何更新 APK？**  
改完后按包类型执行 `npm run cap:sync:debug` 或 `npm run cap:sync`，再在 Android Studio 重新 Build APK。

**Q：调试包和发布包的示例商品不一样？**  
调试同步（`cap:sync:debug`）会打入 `demo-backup.json`；正式发布同步（`cap:sync`）仅在空库时写入参考库三条示例条码，不含演示销售记录。

**Q：备份文件在哪？**  
通过「我的 → 备份数据」唤起分享面板，由用户选择保存位置；文件名形如 `RuralKeeper_backup_YYYYMMDD_HHMMSS.json`。
